# Shuffle · מחולל קומוניקטים

טופס אינטראקטיבי לאמנים. האמן ממלא, לוחץ שליחה, ומגיע אליך קובץ Word מוכן לעריכה במייל.

## איך זה עובד
1. האמן פותח קישור אחד (מחשב או טלפון).
2. ממלא את הטופס, רואה תצוגה חיה של הקומוניקט.
3. לוחץ "שליחה ל-Shuffle".
4. השרת בונה קובץ .docx בעברית RTL ושולח אותו למייל שלך כקובץ מצורף.

## הרצה מקומית
```bash
npm install
cp .env.example .env      # וממלאים את הערכים
npm start                 # http://localhost:3000
```

## פריסה ב-Railway
1. פותחים פרויקט חדש ב-Railway ומחברים את הריפו (או גוררים את התיקייה).
2. Railway מזהה Node אוטומטית ומריץ `npm start`.
3. תחת Variables מגדירים: `RESEND_API_KEY`, `TO_EMAIL`, ואופציונלי `FROM_EMAIL`.
4. מקבלים כתובת ציבורית. זה הקישור שנותנים לאמנים.

## הגדרת שליחת מייל (Resend)
1. נרשמים חינם ב-resend.com ומייצרים API key.
2. שמים אותו ב-`RESEND_API_KEY`.
3. שמים את הכתובת שלך ב-`TO_EMAIL`.
4. לבדיקה ראשונית עובד עם השולח `onboarding@resend.dev`.
   לפרודקשן: מאמתים את `shuffle-x.com` ב-Resend, ומשנים את `FROM_EMAIL` לכתובת משם.

## אבטחה
- שום מפתח או סוד לא נמצא בקוד. הכל דרך משתני סביבה.
- קובץ ה-`.env` לא נכנס ל-git (מוגדר ב-.gitignore).

## הרחבות אפשריות
- פתיחת טאסק ב-ClickUp אוטומטית לכל הגשה.
- שמירת ההגשות ב-SQLite.
- העלאת תמונת עטיפה והטמעתה בקובץ ה-Word.
